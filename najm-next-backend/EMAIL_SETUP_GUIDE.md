# إعداد البريد الإلكتروني للموقع

## المشكلة
الرسائل من نموذج الاتصال لا تصل إلى البريد الإلكتروني.

## الحلول المتاحة

### 1. إصلاح إعدادات Hostinger SMTP (الحالي)
إذا كانت إعدادات Hostinger SMTP صحيحة، تأكد من:

#### التحقق من إعدادات SMTP في ملف `.env`:
```env
SMTP_HOST="smtp.hostinger.com"
SMTP_PORT=587
SMTP_USER="info@najmrush.com"
SMTP_PASS="كلمة المرور الصحيحة"
```

#### خطوات التحقق:
1. **تأكد من صحة كلمة المرور**: قد تحتاج لإعادة تعيين كلمة مرور البريد في لوحة تحكم Hostinger
2. **تحقق من المنفذ**: جرب `587` أو `465`
3. **تأكد من تفعيل SMTP**: في لوحة تحكم Hostinger، تأكد من تفعيل SMTP

### 2. استخدام Resend (مجاني وموثوق)
Resend هي خدمة مجانية لإرسال البريد الإلكتروني.

#### خطوات الإعداد:
1. **إنشاء حساب**: اذهب إلى [resend.com](https://resend.com) وسجل حساباً مجانياً
2. **الحصول على API Key**: من لوحة التحكم، انسخ API Key
3. **تحديث ملف `.env`**:
```env
RESEND_API_KEY="re_1234567890abcdef"
```
4. **تحديث البريد المرسل**: في ملف `src/app/api/send-email/route.js`، غير:
```javascript
from: 'Najm Rush <onboarding@resend.dev>',
```
إلى بريدك الإلكتروني الموثق في Resend.

### 3. استخدام EmailJS (بديل مجاني آخر)
EmailJS هي خدمة مجانية لإرسال البريد الإلكتروني من جانب العميل.

#### خطوات الإعداد:
1. **إنشاء حساب**: اذهب إلى [emailjs.com](https://emailjs.com) وسجل حساباً
2. **إعداد خدمة البريد**: أضف خدمة بريد إلكتروني (Gmail, Outlook, إلخ)
3. **إنشاء قالب**: أنشئ قالب البريد الإلكتروني
4. **الحصول على المفاتيح**:
   - Service ID
   - Template ID
   - Public Key

#### تحديث الكود:
أضف EmailJS إلى المشروع:
```bash
npm install @emailjs/browser
```

ثم أضف الكود إلى صفحة الاتصال.

## الاختبار

### اختبار إرسال البريد:
```bash
node test-email.js
```

### فحص السجلات:
- افتح Developer Tools في المتصفح
- اذهب إلى Console
- ابحث عن رسائل الخطأ المتعلقة بإرسال البريد

## استكشاف الأخطاء

### أخطاء شائعة:

#### 1. "Authentication failed"
- **السبب**: كلمة مرور SMTP خاطئة
- **الحل**: أعد تعيين كلمة مرور البريد في Hostinger

#### 2. "Connection timeout"
- **السبب**: مشكلة في الاتصال بخادم SMTP
- **الحل**: جرب منفذ مختلف أو استخدم Resend

#### 3. "Invalid API key"
- **السبب**: مفتاح Resend API خاطئ
- **الحل**: تحقق من صحة المفتاح في لوحة تحكم Resend

#### 4. "Daily limit exceeded"
- **السبب**: تجاوز الحد اليومي للإرسال
- **الحل**: انتظر حتى يتم إعادة تعيين الحد أو اشترك في خطة مدفوعة

## التوصيات

1. **ابدأ بـ Resend**: أسهل إعداد وأكثر موثوقية
2. **احتفظ بـ SMTP كبديل**: في حالة انقطاع خدمة Resend
3. **راقب السجلات**: تحقق من console للأخطاء
4. **اختبر بانتظام**: تأكد من عمل النظام بإرسال رسائل اختبار

## الدعم

إذا واجهت مشاكل:
1. تحقق من السجلات في الخادم
2. جرب إرسال بريد اختبار مباشرة من خدمة البريد
3. تأكد من إعدادات DNS للدومين
4. اتصل بدعم خدمة البريد المستخدمة
